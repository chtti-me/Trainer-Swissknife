# 教室預約建議模組技術規格 v1.0

## 1. 文件目的

本文件定義「教室預約建議模組」在 `trainer-swissknife` 專案中的技術規格，用於：

- 依據使用者輸入條件，產生「可借用教室建議」。
- 保持「最終預約動作由人員於 `TIS（Training Information System，培訓資訊系統）` 完成」。
- 支援「以 `session（工作階段）` 進行即時查詢」而非長期儲存爬取明細。

---

## 2. v1.0 範圍（Scope，範圍）

### 2.1 目標功能

- 使用者輸入開課條件（日期、時段、院所、教室性質、人數等）。
- 系統透過 `TIS session（工作階段）` 即時查詢教室可用性。
- 回傳前 N 筆教室建議與理由（建議 N=5）。
- 提供「前往 TIS 預約頁面」導引連結。

### 2.2 非目標（v1.0 不做）

- 不直接代替使用者送出最終預約。
- 不做跨天多班最適化排程（Optimization，最佳化）。
- 不做長期教室使用資料倉儲（Data Warehouse，資料倉儲）。

---

## 3. 系統架構

### 3.1 架構角色

- `Next.js（伺服器與前端）`
  - UI（使用者介面）
  - API（應用程式介面）入口
  - 驗證、授權、審計日誌
- `TIS（教室系統）`
  - 真正的查詢與預約來源
- `SQLite（輕量資料庫）`
  - 僅儲存必要設定、審計與建議結果摘要（可選）

### 3.2 資料策略

- v1.0 採「即時查詢優先」。
- 不落地儲存原始 HTML（超文本標記語言）與完整爬取明細。
- 可選擇儲存「建議摘要」與「操作紀錄」以利稽核。

---

## 4. `session（工作階段）` 生命週期規格

### 4.1 平台層生命週期

- 本模組自訂 `session（工作階段）` 有效期：**8 小時**。
- 倒數規則：從建立時間起算 8 小時，到期即失效。
- 更新規則：每次成功查詢可選擇「滑動延長（Sliding Expiration，滑動過期）」或固定不延長；v1.0 建議固定不延長，避免不可預期延展。

### 4.2 與 TIS 失效關係

- 即使本模組 8 小時未到，`TIS session（工作階段）` 仍可能先失效。
- 每次查詢都必須做 `session valid（工作階段有效性）` 檢查。
- 若檢查失敗，回傳 `TIS_SESSION_EXPIRED（TIS 工作階段已過期）`，要求使用者重新提供。

### 4.3 儲存策略（安全）

- 不在前端長期保存明文 `cookie（瀏覽器 Cookie，會話資料）`。
- 後端只保存必要欄位，且需加密（Encryption，加密）或至少遮罩（Masking，遮罩）顯示。
- 支援使用者手動「立即失效」功能（登出 TIS 連線）。

---

## 5. 功能流程（Flow，流程）

1. 使用者進入「教室預約建議」頁面。
2. 輸入或更新 `TIS session（工作階段）`。
3. 系統執行有效性檢查（例如存取可驗證頁面）。
4. 使用者輸入查詢條件：
   - 院所（department）
   - 樓別（building）
   - 教室性質（classroom type）
   - 日期與時段
   - 預估人數
   - 設備需求（可選）
5. 後端查詢 TIS 相關頁面，解析教室可用清單。
6. 套用建議評分規則，產出前 N 筆建議。
7. 顯示建議結果與「前往 TIS 預約」連結。
8. 使用者於 TIS 手動完成最終預約。

## 6. API（應用程式介面）規格（建議）

> 路徑命名採既有專案風格：`src/app/api/.../route.ts`

### 6.1 建立 / 更新 TIS 會話

- `POST /api/classroom/session`

#### Request（請求）

```json
{
  "sessionId": "JSESSIONID=...",
  "source": "manual"
}
```

#### Response（回應）

```json
{
  "ok": true,
  "sessionToken": "local_ref_token",
  "expiresAt": "2026-04-02T10:00:00+08:00",
  "ttlSeconds": 28800
}
```

### 6.2 驗證會話有效性

- `POST /api/classroom/session/validate`

#### Request（請求）

```json
{
  "sessionToken": "local_ref_token"
}
```

#### Response（回應）

```json
{
  "ok": true,
  "tisSessionValid": true,
  "message": "session valid"
}
```

### 6.3 查詢教室建議

- `POST /api/classroom/suggestions`

#### Request（請求）

```json
{
  "sessionToken": "local_ref_token",
  "department": "P",
  "building": "0",
  "date": "2026-04-15",
  "timeStart": "09:00",
  "timeEnd": "12:00",
  "attendees": 30,
  "classroomTypes": ["1", "5"],
  "requiredFeatures": ["projector", "mic"]
}
```

#### Response（回應）

```json
{
  "ok": true,
  "generatedAt": "2026-04-01T16:00:00+08:00",
  "suggestions": [
    {
      "roomId": "PG201",
      "roomName": "G201",
      "building": "綜合大樓",
      "capacity": 40,
      "score": 91,
      "reasons": [
        "時段完全可用",
        "容量符合且有餘裕",
        "設備符合需求"
      ],
      "reserveUrl": "https://tis.cht.com.tw/jap/classroom/ReserveClassroom.jsp"
    }
  ],
  "debug": {
    "candidateCount": 12,
    "excludedCount": 7
  }
}
```

### 6.4 主動失效會話

- `DELETE /api/classroom/session`

#### Request（請求）

```json
{
  "sessionToken": "local_ref_token"
}
```

#### Response（回應）

```json
{
  "ok": true
}
```

## 7. 建議評分規則（Scoring，評分）

v1.0 建議採簡單加權：

- 可用時段符合度：40%
- 容量匹配度：25%
- 設備需求符合度：20%
- 同院所 / 同樓別偏好：10%
- 近期使用穩定性（可選）：5%

### 7.1 排除規則

- 時段衝突直接排除。
- 容量不足直接排除。
- 使用者明確排除的樓別 / 教室代碼直接排除。

---

## 8. 錯誤碼（Error Code，錯誤代碼）

| 代碼 | 說明 | 建議處理 |
|---|---|---|
| `CLASSROOM_BAD_REQUEST` | 參數缺漏或格式錯誤 | 顯示欄位錯誤 |
| `CLASSROOM_SESSION_MISSING` | 未提供 session（工作階段） | 引導重新貼上 |
| `TIS_SESSION_EXPIRED` | TIS 會話失效 | 要求重新登入 TIS 並更新 session |
| `TIS_FETCH_FAILED` | TIS 連線或回應異常 | 顯示重試按鈕 |
| `TIS_PARSE_FAILED` | 頁面結構變更導致解析失敗 | 記錄告警並通知管理者 |
| `NO_SUGGESTION_FOUND` | 無符合條件教室 | 提示放寬條件 |
| `INTERNAL_ERROR` | 伺服器內部錯誤 | 顯示一般錯誤訊息 |

## 9. 資安與合規

### 9.1 敏感資訊處理

- `sessionId（工作階段識別）` 視為敏感資料。
- 僅後端可見明文；前端僅使用 `sessionToken（本地引用權杖）`。
- 日誌中禁止輸出完整 cookie（瀏覽器 Cookie）與個資。

### 9.2 存取控制

- 需登入本系統使用者才能使用建議功能。
- 操作需記錄 `who / when / query condition（誰、何時、查什麼）`，但避免記錄敏感會話值。

### 9.3 連線限制

- 僅允許對白名單網域：`tis.cht.com.tw`。
- 設定單使用者查詢速率限制（Rate Limit，速率限制）。

## 11. 資料模型（可選，v1.0 最小落地）

- `suggestionCount`

- `createdAt`

## 12. v1.0 驗收條件（Acceptance Criteria，驗收標準）

- 可成功接收使用者提供的 `session（工作階段）` 並維持 8 小時平台有效期。
- 當 TIS 先行失效時，能正確回報 `TIS_SESSION_EXPIRED`。
- 使用者可輸入條件並取得至少 1 次有效建議結果（在有可用教室的情況下）。
- 建議結果包含：
  - 教室代碼 / 名稱
  - 可用性判定
  - 分數與理由
  - TIS 預約入口連結
- 不會自動執行最終預約送出。

---

## 13. 後續版本建議（v1.1+）

- 條件模板儲存（常用查詢一鍵套用）。
- 多時段連續性建議（上午 + 下午）。
- AI（人工智慧）補充建議說明文字（例如「為何建議此教室」）。
- 日曆視覺化衝突檢視。
- 增加自動化整合測試，偵測 TIS 頁面欄位變更。
